INTRODUCTION

AI 에이전트란?

스스로 판단하고, 행동하는 AI

단순히 질문에 답하는 AI를 넘어, 목표를 설정하고 계획하고 실행하는 자율적 시스템

  • 01

    환경 인식

    파일, 웹, API 등 외부 환경을 스스로 탐색하고 정보를 수집

  • 02

    자율 판단

    수집한 정보를 바탕으로 다음 행동을 스스로 결정

  • 03

    도구 활용

    코드 실행, 파일 편집, 외부 서비스 호출 등 실제 작업을 수행

  • 04

    반복 학습

    결과를 확인하고 부족하면 스스로 수정하며 목표 달성까지 반복

"매출 자료를 분석해서 매주 월요일 오전 9시에 보고서를 만들어 이메일로 보내줘"
→ AI 에이전트는 이 한 문장으로 전 과정을 자동 실행합니다.

설치 전 확인사항 - WINGET 설치 확인

winget은 Windows 10/11에 기본 내장된 패키지 관리자입니다. 복잡한 설치 과정 없이 명령어 한 줄로 프로그램 설치가 가능합니다.

npm --version 입력 시 스크립트 실행 오류가 발생하나요?

UnauthorizedAccess 또는 Execution_Policies 오류 발생 시 아래 단계를 진행하세요.

  1. 현재 PowerShell 창을 닫습니다.
  2. 시작 메뉴에서 PowerShell을 검색 후 우클릭 → 관리자 권한으로 실행합니다.
  3. 아래 실행 정책 변경 명령어를 입력합니다.
관리자 PowerShell 
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

입력 후 A (모두 예)를 입력하고 엔터를 누른 뒤, 창을 닫고 다시 일반 PowerShell을 엽니다.

CLAUDE.md 역할

에이전트가 대화를 시작할 때마다 자동으로 읽는 지시서이자 정체성을 정의하는 파일입니다.

  • 위치: 프로젝트 폴더 루트 또는 홈 디렉토리 (~/.claude/)
  • 적용: 해당 폴더에서 실행되는 모든 Claude 세션
  • 💡 이름, 역할, 규칙을 적으면 에이전트가 그대로 따릅니다.

Auto Memory

대화 중 중요한 내용을 에이전트가 스스로 판단해 `MEMORY.md`에 자동 기록합니다.

  • 강제 기록: "이걸 기억해줘"라고 명령 시 즉시 저장
  • 효과: 대화가 길어져도 AI가 맥락을 잊지 않도록 방지

파일 제어 (Read / Write / Edit)

READ 파일 읽기

예제 파일 다운로드

텍스트(.txt, .py 등), 이미지(.jpg, .png 등), PDF(최대 100p/20MB) 등 확장자 제한 없이 파일 내용을 읽어 분석합니다. 에이전트가 자동 판단하여 툴을 사용합니다.

  • 예시 1 : ~~경로의 파일 읽어서 파일을 적절한 이름으로 수정해줘
  • 예시 2 : ~파일 열어서 분석 후 정리해줘

WRITE 새 파일 생성

새로운 텍스트 기반 파일을 만들 때 사용합니다. 주의: 기존 파일이 있으면 덮어쓰기(전체 교체)를 합니다.

  • 예시 1 : hello.txt 파일 만들고 자기소개서 작성해줘
  • 예시 2 : 자기소개서.docx 파일 만들고 자기소개서 작성해줘

EDIT 파일 수정

파일 전체를 교체하지 않고 특정 부분만 정밀하게 변경합니다. 수정 전/후 내용을 비교하여 반영하므로 매우 안전합니다.

  • 예시 1 : hello.txt 첫 줄을 '수정된 첫 줄'로 바꿔줘
  • 예시 2 : CLAUDE.md에 항상 존댓말 사용 규칙 추가해줘

AI 에이전트 상세 이해

AI 에이전트는 쉽게 말하면 “목표를 받고, 스스로 판단해서 필요한 작업을 순서대로 실행하는 AI 시스템”입니다.
일반 챗봇이 “질문 → 답변” 중심이라면, AI 에이전트는 한 단계 더 나아가서 목표 달성을 위해 계획하고, 도구를 사용하고, 결과를 확인하면서 작업을 이어가는 구조라고 볼 수 있습니다.

1. AI 에이전트의 기본 작동 흐름

AI 에이전트는 보통 아래 순서로 작동합니다.

목표 입력 목표 이해 작업 계획 도구 선택 작업 실행 결과 확인 수정/반복 최종 결과

예를 들어 "이번 주 제주도 여행 계획 짜줘."라고 했을 때 일반 AI는 바로 코스를 추천하지만, AI 에이전트는 다릅니다.

  1. 여행 날짜 확인
  2. 날씨 검색
  3. 항공권/숙소 조건 확인
  4. 관광지 후보 정리
  5. 이동 동선 계산
  6. 일정표 작성
  7. 예산 정리

2. AI 에이전트를 구성하는 핵심 요소

① 목표 인식

"보고서 작성해줘"라는 요청을 받으면 주제, 독자, 필요 자료, 형식, 근거 등을 구체적으로 파악합니다.

② 계획 수립

목표를 작은 작업 단위로 나눕니다. (시장 규모 조사 → 트렌드 정리 → 사례 수집 → 구조 작성 → 다듬기)

③ 도구 사용

상황에 맞는 도구를 골라 작업합니다. (웹 검색, 캘린더, 이메일, 문서 도구, 코드 실행, 이미지 생성 등)

④ 실행과 관찰

작업을 실행한 뒤 결과가 충분한지, 출처가 믿을만한지 등을 판단합니다.

⑤ 반복 개선

한 번에 끝내지 않고 부족한 점을 확인하여 결과물을 더 좋게 다듬는 반복 과정을 거칩니다.

3. 일반 AI vs AI 에이전트

구분 일반 AI 챗봇 AI 에이전트
역할질문에 답변목표를 수행
방식즉시 응답 중심계획 → 실행 → 검토
도구 사용제한적이거나 없음적극적으로 사용
작업 범위단일 답변여러 단계 작업
예시"설명해줘""자료 찾아서 보고서 만들어줘"

일반 AI = 똑똑한 답변자 | AI 에이전트 = 일을 처리하는 디지털 작업자

4. AI 에이전트 활용 예시

여행 에이전트

"3박 4일 부산 여행 계획 짜줘"

  • 날짜/날씨 확인
  • 숙소 위치 고려
  • 이동 동선 및 일정표 작성
업무 자동화 에이전트

"회의록 정리해서 메일 보내줘"

  • 핵심 내용 요약 / 할 일 추출
  • 메일 초안 작성 및 발송
쇼핑 에이전트

"10만 원 이하 무선 키보드 추천해줘"
